Within the past few years I have worked on numerous open source projects, both in teams and alone, but here is a list of my favorite projects:

Lithium

Technologies: Java 8, netty, peer-to-peer networking, JavaFX, Cryptography (AES, RSA and Diffie-Hellman)

Lithium is my latest big project in a long series. Its mission is to provide a de-centralized secure and simple to use crypto-messenger which protects the privacy of its users. Without doubt this is one of my favorite projects clocking in with just about two weeks of work for a complete working prototype along with research on the subject of peer-to-peer communication.

Website »

Ikarus

Technologies: PHP 5.4, HTML 5, CSS 3, JavaScript

Ikarus will always have a special place in my heart as it has been my first bigger attempt of designing a working fully-fledged PHP framework which was capable of adjusting to various environments. While it has ended a couple of years ago with me moving on to Java development, it is still around and very much capable of running applications to this day.

Website »

Powersweeper

Technologies: Java 8, Selenium, JavaScript

Created out of a joke (in short "We wasted so much time on this stupid Minesweeper game, let's at least program a bot for it which kicks ass"), the Powersweeper turned out to be a great work of uncoordinated collaboration (interfaces were declared as a guideline while the implementation was created around it) which still works rather well today ... and is probably entirely against the rules of the website it was designed to run on.

Website »

Candle

Technologies: Java 8, ANTLR

The Torchmind Configuration Syntax (or Candle for short - which does not seem to have any relation to its full name), is an attempt to simplify server side configurations for less tech savvy users as it aims to stick as close to a natural approach of expressing objects as possible.

Website »

UPM

Technologies: Java 8, XSD

The Universe Package Manager is a concept I recently experimented around with and while it is currently only in a draft stage (e.g. most of its descriptors are fully designed buts its full implementation is still on hold as I experiment around with various concepts) it still provides an interesting starting point for the future of modular applications.

Website »

Mango

Technologies: Java 8

Mango is so far one of the biggest collections of entirely unrelated implementations for Java I've written. It provides countless methods of simplifying syntax to improve maintainability of Java code and regularly receives new features as I realize that I'm writing more code than I should be.

Website »

Padlock

Technologies: Java 8, Cryptography (RSA)

While it was never used as it didn't have a use with the static web application approach my customers required at the time, Padlock provided (and still provides today) a secure method of implementing session less authentication with web servers. The library was mostly aimed at rich web applications and desktop clients which cannot rely on cookies.

Website »

Bukkit Plugin Annotations

Technologies: Java 8, Annotation Processing

Inspired by an existing solution, which just wasn't available in any maven repository nor utilized Java 8's capabilities of repeating annotations, the Bukkit Plugin annotations provide a great method of generating plugin descriptors for the Bukkit Minecraft Server API.

Website »
